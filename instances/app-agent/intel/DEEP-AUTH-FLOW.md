# Deep Dive: Authentication & RBAC

**Traced:** 2026-06-18 (S03)  
**Slug:** `authentication`

## Stack

- **Module:** `nuxt-auth-utils` (opt-in per app via `modules` in `nuxt.config.ts`)
- **Session:** Encrypted cookie via `NUXT_SESSION_PASSWORD` in `.env`
- **OAuth:** `server/routes/auth/[provider].get.ts` per app (popup flow)

## Server middleware (`core/server/middleware/03.auth.ts`)

`defineFeatureHandler('authentication', ...)` — runs on every request when middleware is active:

1. Reads `useRuntimeConfig().session?.password`
2. **If no password:** sets `event.context.auth = { loggedIn: false, user: null, role: 'public' }` and returns (graceful no-op)
3. **If configured:** calls `getAuthUser(event)` → populates `event.context.auth`

```typescript
event.context.auth = {
  loggedIn: !!user,
  user,
  role: user?.role || 'public'  // 'public' | 'registered' | 'admin'
}
```

Middleware is **additive across layers** — cannot be overridden, only extended.

## Client composable (`core/app/composables/useAuth.ts`)

`defineFeatureComposable('authentication', ...)`:

- If `useUserSession` unavailable (module not loaded) → returns safe defaults (all false/null)
- Otherwise exposes:
  - `loggedIn`, `user`, `role`, `isAdmin`, `isRegistered`
  - `login(provider)` → `openInPopup('/auth/${provider}')`
  - `logout()` → `clear()` + `navigateTo('/')`

## Organization defaults (`organization/app/app.config.ts`)

Auth **UI config** (not security) is centralized:

- `auth.providers` — Google, GitHub labels/icons
- `auth.login` / `auth.signup` — form fields, SEO, copy
- Demos read these via `useAppConfig()` — issue #114 targets reducing hardcoding here

## Per-app opt-in

| App | `nuxt-auth-utils` | Notes |
|-----|-------------------|-------|
| `apps/chat` | ✅ | Full OAuth + session |
| `demos/chat` | ✅ | Demo variant |
| `demos/saas`, `landing` | ✅ | Auth pages use `auth` config |
| `demos/dashboard` | Partial | User menu, not full auth flow |
| `core/docs`, `control` | Varies | Control has agent, not user auth |

## RBAC model

Three roles: `public` → `registered` → `admin`.  
Server routes should check `event.context.auth.role` — no centralized route guard macro yet.

## Known issues

- **#111** — `CoreUserMenu` login buttons truncated (UI bug)
- Auth is **partially productized** — no billing, no multi-tenant org model
- `temp-product-context.md` vision (full auth/billing) not built

## Security notes

- Session password must be 32+ chars — generated by launcher on demo copy
- OAuth secrets: `NUXT_OAUTH_*` per provider in app `.env`
- Never commit `.env` — use `# Secrets` manifest + `multi-encrypt`
