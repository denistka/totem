## 2. ПРОБЛЕМА

Мільйони людей щодня приймають фінансові рішення наосліп — не тому що вони нерозумні, а тому що ніхто ніколи не пояснював їм правила гри.

**Розрив у доступі:**

| Багаті | Всі інші |
|---|---|
| Особистий бухгалтер | Google |
| Фінансовий радник | Реклама банків |
| Податковий адвокат | Поради друзів |
| $500–2000/міс | Роки помилок |

**Системна проблема:** Банки заробляють на плутанині. Фінансова безграмотність — не особиста проблема, а системна вразливість країни.

## 3. РІШЕННЯ

Персональний AI який поєднує три ролі в одному голосі:

### 🏦 Банкір

- Допомагає з податками для ФОП
- Знаходить де можна скоротити витрати прямо зараз

- Розуміє фінансовий патерн конкретної людини
- Будує план виходу з кризи по кроках
- Прогнозує майбутнє при різних сценаріях
- Навчає через власний досвід користувача

## 4. ГОЛОС ПРОДУКТУ

Не робот. Не консультант. **Старший друг який розбирається в фінансах.**

```
❌ "Ваша заборгованість складає 63 000 грн. Рекомендуємо погасити."

✅ "Окей, 63к на моно після пільгового — це боляче.
    Зараз ти платиш 94 ₴ на день просто за те що борг існує.
    Ось що можна зробити вже сьогодні щоб це зупинити."
```

**Три покоління — один голос, різна подача:**

| Вік | Стиль | Страх |
|---|---|---|
| 20–35 | Швидко, конкретно, без зайвого | "Не хочу думати про гроші" |
| 35–55 | Надійно, з поясненнями | "Боюсь втратити що є" |
| 55+ | Повільно, з повагою, без жаргону | "Мене обманять" |

## 8. ТЕХНІЧНИЙ СТЕК

### Frontend
```
Framework:    Vue 3 (Composition API)
Build:        Vite
PWA:          vite-plugin-pwa
State:        Pinia
HTTP:         Axios
Composables:  @vueuse/core
Стиль:        CSS Variables (без UI-бібліотек — власний дизайн)
```

### Backend
```
Runtime:      Node.js 20+
Framework:    Express.js
AI:           Anthropic Claude API (claude-sonnet-4-20250514)
Realtime:     WebSocket (ws) — вже є в наробках
Database:     PostgreSQL via Supabase
Auth:         Telegram initData verification
Push:         Web Push (VAPID) + Telegram бот
```

### Infrastructure
```
Frontend:     Vercel (безкоштовно, auto-deploy з Git)
Backend:      Railway (від $5/міс)
Database:     Supabase (безкоштовний tier достатній для MVP)
Domain:       bankir.ai
CDN:          Vercel Edge автоматично
```

### Telegram
```
Bot:          Telegraf.js
Mini App:     React-сумісний (той самий Vue build)
Верифікація:  HMAC-SHA256 initData
```

### Моніторинг (після запуску)
```
Errors:       Sentry (безкоштовний tier)
Analytics:    Plausible або Umami (privacy-first)
Uptime:       UptimeRobot
```

## 10. СХЕМА БД (MVP)

```sql
-- Користувачі
CREATE TABLE users (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  telegram_id BIGINT UNIQUE,
  scenario TEXT,           -- debt | budget | fop | goal | general
  age_group TEXT,          -- young | mid | senior
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- Розмови (пам'ять AI)
CREATE TABLE conversations (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES users(id),
  messages JSONB NOT NULL DEFAULT '[]',
  updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- Борги
CREATE TABLE debts (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES users(id),
  bank TEXT,
  amount NUMERIC,
  monthly_rate NUMERIC,    -- 0.035 = 3.5%
  start_date DATE,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- Тарифи банків (оновлюємо вручну на старті)
CREATE TABLE bank_rates (
  bank TEXT NOT NULL,
  operation TEXT NOT NULL, -- p2p | cash | iban | international
  rate NUMERIC DEFAULT 0,
  min_fee NUMERIC DEFAULT 0,
  note TEXT,
  updated_at TIMESTAMPTZ DEFAULT NOW(),
  PRIMARY KEY (bank, operation)
);

-- Push підписки
CREATE TABLE push_subscriptions (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES users(id),
  endpoint TEXT NOT NULL,
  keys JSONB NOT NULL,
  created_at TIMESTAMPTZ DEFAULT NOW()
);
```
