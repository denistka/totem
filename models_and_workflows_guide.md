# Гайд по моделям и режимам разработки Antigravity

Этот документ поможет эффективно распределять задачи между ИИ-моделями и выбирать правильный поток работы (workflow) для проекта **Navitrack**.

---

## 1. Ранжирование моделей (IQ vs Speed)

Ниже представлен рейтинг основных моделей, доступных в интерфейсе, от самых мощных до самых быстрых.

| Модель | Роль | Сильные стороны | Когда использовать |
| :--- | :--- | :--- | :--- |
| **Claude Opus 4.6 (Thinking)** | **Архитектор** | Глубочайшее «мышление», решение сложнейших алгоритмических задач. | Проектирование систем с нуля, «невозможные» баги. |
| **Claude Sonnet 4.6 (Thinking)** | **Senior Developer** | Точное следование инструкциям, идеальный баланс ума и скорости. | Разработка фич «под ключ», сложная бизнес-логика. |
| **Gemini Pro / GPT-OSS 120B** | **Middle+ All-rounder** | Стабильность, работа с большим контекстом проекта. | Надежное написание кода, ревью, общие задачи. |
| **Gemini Flash** | **Ассистент / Speedster** | Мгновенная реакция, высокая скорость работы. | UI компоненты, верстка, тесты, документация. |

---

## 2. Сумма смыслов: Режимы работы

Режим определяет способ взаимодействия с кодом.

### ⚡ Fast Mode (Реактивный)
*   **Смысл:** Прямое действие без лишних слов.
*   **Логика:** Вы просите — я сразу меняю код.
*   **Эффективность:** Максимальна для задач, где результат очевиден и локализован в 1-2 файлах.

### 📋 Planning Mode (Стратегический)
*   **Смысл:** Сначала чертеж — потом стройка.
*   **Логика:** Исследование -> План (Implementation Plan) -> Согласование -> Реализация.
*   **Эффективность:** Незаменима для фич, затрагивающих несколько модулей, API или глобальный стейт.

---

## 3. Матрица принятия решений (Рекомендованные пары)

Чтобы не тратить лишнее время, используйте эти связки:

1.  **«Нужно поправить баг в логике на фронте»**
    *   **Модель:** Claude Sonnet 4.6 (Thinking)
    *   **Режим:** Fast

2.  **«Нужно внедрить новую систему отслеживания истории объектов»**
    *   **Модель:** Claude Sonnet 4.6 (Thinking) или Opus
    *   **Режим:** Planning

3.  **«Нужно накидать UI-компонентов по макету»**
    *   **Модель:** Gemini Flash
    *   **Режим:** Fast

4.  **«Нужно проверить весь проект на наличие дыр в безопасности»**
    *   **Модель:** Gemini Pro или GPT-OSS 120B
    *   **Режим:** Planning

---

## 4. Жизненный цикл задачи (Best Practice)

Для достижения идеального результата в Navitrack рекомендуется такой цикл:

1.  **Исследование (Planning Mode + Pro/Sonnet):** Анализ влияния фичи на проект.
2.  **Прототип (Fast Mode + Sonnet):** Написание основного скелета логики.
3.  **Обустройство (Fast Mode + Flash):** Стилизация (CSS/TSX) и мелкие UI детали.
4.  **Закрытие (Fast Mode + Flash):** Написание тестов и документации.
